# 需求收集问题模板

> 用于在需求收集阶段，逐个询问用户，确保需求完整。

---

## 使用说明

1. **逐个询问** — 每次只问一个问题，等用户回答后再问下一个
2. **记录答案** — 每个问题的答案都要记录下来
3. **不自己发挥** — 用户没说的功能，不要自己添加
4. **追问细节** — 用户回答模糊时，追问具体细节

---

## 问题列表

### 第一部分：项目基本信息

#### Q1. 项目背景
**问题**：这个项目是做什么的？解决什么痛点？

**追问**：
- 现在没有这个系统时，用户是怎么解决问题的？
- 为什么现在需要做这个系统？

---

#### Q2. 目标用户
**问题**：谁会使用这个系统？（比如：C端用户、企业客户、内部员工等）

**追问**：
- 他们的年龄、职业、技术水平大概是什么样的？
- 预估有多少用户？

---

#### Q3. 平台类型
**问题**：这个系统要支持哪些平台？

- [ ] Web 网页
- [ ] Android App
- [ ] iOS App
- [ ] 微信小程序
- [ ] 后台管理系统
- [ ] 其他：____

**追问**：各平台优先级是什么？先做哪个？

---

### 第二部分：用户角色与权限

#### Q4. 用户角色
**问题**：系统需要支持哪些角色？每种角色做什么？

**选项**：
- 管理员（后台管理）
- 普通用户（使用核心功能）
- 商家/供应商（如有）
- 客服（如有）
- 其他：____

---

#### Q5. 权限管理
**问题**：不同角色之间的权限如何划分？是否需要区分等级？

**追问**：
- 管理员能看到所有数据吗？
- 普通用户之间数据隔离吗？

---

### 第三部分：核心功能

#### Q6. 核心业务流程
**问题**：用户使用系统的**主要流程**是什么？请描述一个典型用户的一天。

**追问**：
- 用户打开 App/网站后，第一步做什么？
- 第二步做什么？
- 完成什么就算成功了？

---

#### Q7. 主要功能清单
**问题**：系统需要哪些**具体功能**？

请逐个描述，例如：
- 用户注册登录
- 查看列表
- 下单/预约
- 支付
- 消息通知
- ...

**追问**：每个功能的核心是什么？用户在这个功能里做什么？

---

#### Q8. 功能优先级
**问题**：这些功能里，哪些是**必须有**的（MVP）？哪些是**后面再加**的？

**格式**：
| 优先级 | 功能 |
|--------|------|
| P0（必须有） | |
| P1（第二阶段） | |
| P2（后续迭代） | |

---
