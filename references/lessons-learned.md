# Lessons Learned（经验教训）

> 每个项目结束后，记录踩坑记录和解决方案，形成知识积累。

---

## 核心理念

```
踩过的坑 = 未来的护城河
```

每次项目中发现的问题、犯过的错误、找到的解决方案，都应该记录下来，供后续项目参考。

## 分类参考

### Backend（后端）

| 常见教训 | 说明 |
|----------|------|
| API 版本管理 | 接口变更要兼容旧版本 |
| 参数校验 | 永远不要相信前端传来的参数 |
| 事务边界 | 合理控制事务范围，避免长事务 |
| 异常处理 | 统一异常处理，不要吞掉异常 |
| 日志规范 | 结构化日志，便于排查问题 |
| 缓存穿透 | 空值也要缓存，防止击穿 |
| SQL 注入 | 使用参数化查询 |

### Frontend（前端）

| 常见教训 | 说明 |
|----------|------|
| 状态管理 | 大型状态用状态库，小型用 useState |
| 内存泄漏 | 组件卸载时要清理副作用 |
| 首屏加载 | 代码分割、懒加载 |
| 表单校验 | 前端校验不能替代后端校验 |
| 兼容性问题 | 测试多浏览器、多设备 |
| 安全 | XSS、CSRF 防护 |

### Database（数据库）

| 常见教训 | 说明 |
|----------|------|
| 索引设计 | 不是越多越好，索引也有代价 |
| 数据迁移 | 禁止在线表结构变更，必须用脚本 |
| 备份恢复 | 定期演练，恢复流程要熟练 |
| SQL 性能 | 避免全表扫描、N+1 查询 |
| 数据清理 | 定期清理日志、无用数据 |

### DevOps（运维）

| 常见教训 | 说明 |
|----------|------|
| 环境一致 | 开发/测试/生产环境要一致 |
| 密钥管理 | 密钥不上传代码仓库 |
| 回滚方案 | 每次部署必须有回滚方案 |
| 监控告警 | 重要指标必须监控 |
| 灾备方案 | 有异地灾备方案 |

### Management（管理）

| 常见教训 | 说明 |
|----------|------|
| 需求变更 | 评估影响再接受变更 |
| 排期估算 | 估算要留 Buffer |
| 技术选型 | 新技术要充分评估风险 |
| 代码审查 | 必须有 Code Review |
| 文档同步 | 代码变更新要及时更新文档 |

## 模板示例

### 示例 1：后端 - API 版本管理

```markdown
# Lesson: API 大版本变更导致线上故障

## 基本信息
| 字段 | 值 |
|------|-----|
| 日期 | 2026-04-03 |
| 项目 | 旧衣服回收系统 |
| 发现者 | Backend Agent |
| 类别 | backend |
| 严重程度 | Critical |

## 问题描述
API v2 接口返回格式与 v1 不兼容，导致已发布的 iOS App 全部崩溃。

## 影响
- 线上用户无法使用 App
- 紧急回滚到 v1

## 根本原因
- 发布 v2 时没有考虑向后兼容
- 没有版本管理策略

## 解决方案
短期：回滚到 v1
长期：
- API 版本管理策略：vN 与 vN+1 必须兼容
- 新版本先灰度，再全量

## 预防措施
1. API 变更必须通过 API Review
2. 大版本发布前必须有兼容性测试
3. 制定 API 版本管理策略文档

## 相关文档
- API_VERSION_STRATEGY.md

## 标签
#api #version #compatibility #rollback
```

### 示例 2：前端 - 内存泄漏

```markdown
# Lesson: React 组件内存泄漏导致页面卡顿

## 基本信息
| 字段 | 值 |
|------|-----|
| 日期 | 2026-04-02 |
| 项目 | 旧衣服回收系统 |
| 发现者 | Frontend Agent |
| 类别 | frontend |
| 严重程度 | Medium |

## 问题描述
订单列表页面长时间使用后，页面越来越卡顿，最终崩溃。

## 影响
- 用户体验下降
- 部分用户反映页面卡顿

## 根本原因
- 定时器没有在组件卸载时清除
- 订阅事件没有取消订阅

## 解决方案
```javascript
useEffect(() => {
  const timer = setInterval(refreshOrders, 5000);
  return () => {
    clearInterval(timer);  // 组件卸载时清理
  };
}, []);
```

## 预防措施
1. useEffect 必须有清理函数
2. 组件卸载前要清理所有副作用
3. 使用 ESLint 规则检查

## 标签
#react #memory-leak #cleanup
```
